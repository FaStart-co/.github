# Contributing to FaStart

Thank you for your interest in FaStart.

Most of FaStart's core product repositories are currently private while the platform is in active development. Public repositories in this organization may include documentation, templates, SDKs, examples, integrations, or developer utilities.

## Before You Contribute

For public repositories, please start by checking the repository's own README, issues, and contribution notes. Repository-specific instructions always take priority over this default guide.

## Contribution Workflow

1. Open an issue before starting larger changes.
2. Keep pull requests focused and easy to review.
3. Include context, screenshots, examples, or reproduction steps when useful.
4. Update documentation when behavior changes.
5. Respect existing code style, naming, architecture, and project conventions.

## Pull Request Expectations

A strong pull request should include:

- A clear description of the change
- The reason behind the change
- Testing or verification notes
- Screenshots or recordings for UI changes
- Linked issues or context when relevant

## Community Standards

All contributors are expected to follow our [`CODE_OF_CONDUCT.md`](./CODE_OF_CONDUCT.md).
